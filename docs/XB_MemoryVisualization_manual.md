## 1. 功能概览

硬件监控面板是一个**实时、可拖拽、可缩放**的系统资源监控窗口，在 ComfyUI 界面右下角自动显示。

### 核心功能

| 功能 | 说明 |
|------|------|
| **VRAM 显存监控** | 实时显示 GPU 显存使用量、分段占比（模型/Torch/缓存/其他） |
| **RAM 内存监控** | 系统内存、进程内存、固定内存、Python 占用 |
| **GPU 状态** | 利用率、温度、功耗（NVIDIA 全支持 / AMD 需 LibreHardwareMonitor） |
| **CPU 状态** | CPU 使用率、型号识别 |
| **磁盘监控** | 磁盘读写速率、各分区剩余空间 |
| **页面文件** | 虚拟内存使用情况 |
| **模型详情** | 已加载模型列表、每个模型的 VRAM/内存分段占用 |
| **历史图表** | VRAM 使用量趋势图（面积图/柱状图/K线图/点阵图） |
| **快捷操作** | 一键卸载模型、清除缓存、执行/中断工作流 |

## 2. 安装与依赖

### 自动安装

面板**无需放置节点即可自动加载**。ComfyUI 启动后，监控窗口自动出现在界面右下角。

### 必备依赖

```bash
pip install psutil pythonnet
```

| 依赖 | 用途 | 必需 |
|------|------|------|
| `psutil` | CPU/内存/磁盘/进程信息 | ✅ 是 |
| `pythonnet` | .NET 桥接，驱动 LibreHardwareMonitor | ✅ 是 |
| `LibreHardwareMonitorLib.dll` | GPU 传感器读取（已内置） | ✅ 是 |
| `nvidia-ml-py` | NVIDIA GPU 精准显存数据 | ⚠️ 可选 |

> `LibreHardwareMonitorLib.dll` 已内置在 `XB_ToolBox\` 目录中，无需额外下载。

### NVIDIA 用户（可选）

```bash
pip install nvidia-ml-py
```

安装后可获得设备级显存数据（而非进程级），Windows WDDM 下尤其有用。

## 3. 快速上手

### 3.1 启动面板

1. 启动 ComfyUI
2. 面板**自动出现**在界面右下角
3. 如果没有看到，检查 ComfyUI 控制台是否有报错

### 3.2 面板操作

| 操作 | 方法 |
|------|------|
| **移动面板** | 拖拽标题栏 |
| **调整大小** | 拖拽面板左/右/下边缘或右下角 |
| **折叠/展开** | 点击标题栏 `−` / `+` 按钮 |
| **弹出窗口** | 点击 `⤤` 按钮（画中画模式） |
| **运行工作流** | 点击 `▶` 按钮 |
| **取消运行** | 运行中点击 `■` 按钮 |
| **打开菜单** | 右键点击面板任意位置 |

你也可以将 `XB 📊 硬件监控面板` 节点拖入工作流：
- 节点本身**不产生任何输出**
- 执行节点时会在控制台打印当前 VRAM 状态
- 主要用作工作流中的"书签"标记

| `⤤` | 弹出画中画窗口 |
| `−` / `+` | 折叠/展开面板 |

### 4.2 内容区

#### 内存行
```
内存  32.5GB|64.0GB
████████░░░░░░░░░░░░░░░░░░░░
■ 固定 9.9GB  ■ 已加载 0KB  ■ Python 11.3GB  ■ 其他 11.3GB
```

- **固定内存 (Pinned)**：用于 GPU 快速传输的锁页内存
- **已加载 (Loaded)**：已从磁盘加载到 RAM 的模型数据
- **Python**：Python 进程自身占用
- **其他**：系统及其他进程占用

#### 显存行
```
显存 (AMD Radeon RX 7900 XTX)  9.0GB|15.9GB
████████████████░░░░░░░░░░░░░░░░
■ 模型 5.7GB  ■ Torch 2.1GB  ■ 缓存 0.5GB  ■ 其他 0.7GB
峰值: 12.3GB  缓存: 1.2GB  GPU 45%  55°C  9W  ● 运行中
```

- **模型 (Models)**：已加载模型占用的 VRAM
- **Torch**：PyTorch 活跃分配
- **缓存 (Cache)**：PyTorch 缓存池（可释放）
- **其他 (Other)**：驱动及其他进程占用

底部信息行：
| 项目 | 说明 |
|------|------|
| `峰值` | 自面板启动以来的 VRAM 最大使用量 |
| `缓存` | PyTorch 缓存池大小 |
| `GPU XX%` | GPU 核心利用率（点击可切换图表上的 GPU 曲线） |
| `XX°C` | GPU 温度 |
| `XX W` | GPU 功耗 |
| `● 运行中` / `● 空闲` | 工作流执行状态 |

#### 页面文件行
```
页面文件  2.1GB|16.0GB
████████░░░░░░░░░░░░░░░░░░░░░░
```
仅当系统启用了页面文件时显示。

## 9. AMD GPU 用户须知

### 支持的 GPU

| GPU 系列 | 架构 | 利用率 | 温度 | 功耗 | VRAM |
|----------|------|--------|------|------|------|
| RX 7000 系列 | RDNA 3 | ✅ | ✅ | ✅ | ✅ |
| RX 6000 系列 | RDNA 2 | ✅ | ✅ | ✅ | ✅ |
| RX 5000 系列 | RDNA 1 | ✅ | ✅ | ✅ | ✅ |
| Vega / Radeon VII | GCN | ✅ | ✅ | ✅ | ✅ |

### 监控原理

AMD GPU 监控通过 **LibreHardwareMonitor** 实现（一个跨厂商的开源硬件监控库）。原理是直接读取 GPU 的 SMU（System Management Unit）传感器数据，与 AMD Adrenalin 驱动面板使用相同的底层接口。

### 如果数据不显示

1. 确认 `XB_ToolBox\LibreHardwareMonitorLib.dll` 存在
2. 确认已安装 `pythonnet`：`pip install pythonnet`
3. 尝试**以管理员身份**运行 ComfyUI
4. 查看 ComfyUI 控制台是否有 `aimdo-viz: LibreHardwareMonitor` 相关日志

### 功耗显示的差异

AMD 功耗显示为 **GPU Package**（整个显卡的总功耗），单位是瓦特 (W)，没有功率上限对比。这与 NVIDIA 的显示格式略有不同（NVIDIA 有 `功耗/上限` 格式）。

## 10. 常见问题

### Q: 面板没有自动出现？

A: 检查 ComfyUI 控制台是否有 JavaScript 报错。常见原因：
- `pythonnet` 未安装
- `LibreHardwareMonitorLib.dll` 缺失
- JS 文件加载失败（检查 `js/xb_memory_viz.js` 是否存在）

### Q: GPU 利用率/温度/功耗都显示"无数据"？

A: 
- **NVIDIA**：安装 `nvidia-ml-py` 并重启
- **AMD**：确认 `pythonnet` 已安装，查看控制台日志

### Q: 面板太大/太小？

A: 右键 → **缩放** 调整面板大小，或右键 → **字号** 仅调整文字大小。

### Q: 如何让面板固定在顶栏？

A: 拖拽面板到顶部 ComfyUI 操作栏的"停靠左侧"或"停靠右侧"区域释放，或右键 → 停靠到顶栏。

### Q: 图表不更新？

A: 检查轮询间隔设置（右键 → 轮询间隔）。确保未将面板拖拽到历史数据中（右侧显示"实时"即为实时模式）。

### Q: 节点放到工作流中有用吗？

A: 面板**不需要**节点即可工作。节点的作用是：
- 作为工作流书签
- 执行时在控制台打印 VRAM 状态
